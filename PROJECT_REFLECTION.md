Project Reflection

What we Learned :
Working on this project taught us things that go well beyond what any 
textbook could cover. We came in knowing the basics of Python and left 
with a genuinely different understanding of what data science actually 
looks like in practice.

The most surprising lesson was how much time real data work takes before 
you even get to the interesting parts. Cleaning five datasets, handling 
missing values, fixing date formats, and making sure everything merges 
correctly took longer than we expected. We learned that messy data is 
not an exception in the real world — it is the rule. This matters because 
any model or visualization built on poorly cleaned data will produce 
misleading results no matter how sophisticated the analysis is.

We will carry this understanding into every data project we work on going 
forward. Before touching any model or chart, clean the data first and 
understand it deeply.

---

What Changed in How We See Things:
Before this project we thought of climate change mostly in terms of 
headlines and news reports. Working directly with the temperature anomaly 
data changed that. Seeing the numbers climb year by year in a dataset 
we cleaned ourselves made the trend feel real in a way that reading 
about it never did.

We also came to see machine learning differently. Before starting, we 
imagined it as something almost magical — you feed in data and get 
back answers. After building our own models we understand that the 
quality of predictions depends entirely on the quality and relevance 
of the input data, and that every model has limitations that need to 
be stated honestly.

What we know now that we did not know before: a high R-squared score 
does not mean a model is right. It means the model fits the historical 
data well. Whether that pattern continues into the future is a separate 
question entirely.

---

Skills Developed :
**Critical Thinking and Problem Solving**
Almost every day of this project involved hitting an unexpected problem 
and figuring out how to fix it. Git merge conflicts, missing Python 
packages, kernel errors, and file path issues all required us to stop, 
diagnose the problem carefully, and find a solution without breaking 
something else. This kind of debugging thinking is something we 
developed gradually over the week and is probably the most transferable 
skill we took away.

**Teamwork and Collaboration**
Working on the same codebase through GitHub forced us to communicate 
clearly and constantly. We learned that good collaboration is not just 
about dividing tasks — it is about making sure your work integrates 
cleanly with someone else's. We made mistakes early on by not pulling 
before pushing and by running each other's notebooks. These mistakes 
taught us more about version control than any tutorial would have.

**Technology**
We went from having Git installed but barely used to managing a shared 
repository with branching, merging, conflict resolution and a clean 
commit history. On the Python side we moved from knowing pandas 
superficially to using it for real multi-dataset merges, time series 
handling and feature engineering for machine learning.

**Written Communication**
Writing the README and project summary pushed us to explain technical 
work in plain language. Describing what a Random Forest model does to 
someone who has never heard of it — without being condescending or 
overly technical — is genuinely difficult. We rewrote those sections 
several times before they felt right.

**Professionalism and Work Ethic**
We treated this project like a real professional assignment. That meant 
consistent daily commits, meaningful commit messages, clean folder 
structure, and making sure the project works for someone who has never 
seen it before. These habits did not come naturally at first but by the 
end of the week they felt automatic.

**Self Awareness**
This project revealed our individual strengths clearly. One of us is 
more comfortable with the analytical and modeling side while the other 
approaches problems more visually and thinks about how to present 
findings clearly. Recognizing this made us better at dividing work in 
a way that played to both our strengths.

---

Global Perspective :
Climate change is not a local problem and this project made that 
concrete. Looking at CO2 emissions data country by country, watching 
temperature anomalies rise across every region simultaneously, and 
seeing how a carbon pricing policy in Europe sends ripples through a 
market that affects global emissions decisions — all of this gave us 
a much more connected view of the problem than we had before.

Data science applied to global challenges like this one is not neutral. 
The way you frame a visualization, which countries you highlight, which 
time period you choose — these decisions shape what story the data tells. 
We became more aware of this responsibility as the project went on.

---

Looking Forward :
This project confirmed for both of us that pursuing a master's degree 
in AI or Data Analytics is the right direction. Not because it was 
easy — it was not — but because the moments when something finally 
worked, when a chart revealed a pattern we had not expected, or when 
a model's predictions lined up closely with reality, were genuinely 
exciting. That feeling is what we want more of.

We plan to build on this foundation by learning more about deep learning 
for time series forecasting, exploring interactive visualization tools 
like Plotly and Streamlit, and contributing to open source climate 
data projects. This project was a starting point, not an endpoint.
